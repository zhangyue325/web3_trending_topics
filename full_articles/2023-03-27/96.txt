**02**

**孵化器**

**1**►

**孵化器是什么**

“孵化器”这个概念，是跟着互联网行业的发展而兴起的。孵化器旨在帮助创业团队克服早期发展中遇到的各种困难和障碍，协助创始人将创意转化为可扩展、可落地的业务。在互联网行业中，孵化器主要提供产品设计和技术开发等方面的指导，帮助早期团队加速产品验证的过程。此外，大部分的创业孵化器还提供线下办公场地、赛道资源，以及帮助寻找合伙人、融资等等。

  


**2**►

**Web3 孵化器 vs 传统孵化器**  


那么 Web 3 的孵化器与传统孵化器有什么不同？

  


目前 **Web3 孵化器的形式很多**，如有的以 Dao 形式存在，有的以中间化的方式去运作等等。整体上看， **Web3 比互联网的早期项目孵化器更原始**，更像一个非常初始的创始人社区。大家都对这行业非常有热情，想做自己的项目，然后聚在这个 community 里面去讨论、解决一些行业赛道里真实的痛点和需求。

  


至于 Web 3 的孵化器与传统 Web 2 的具体区别：

  


一是 **Web 3 孵化器的孵化周期比传统孵化器短**，一般会在一两个月，甚至是几周的时间内完成。

  


二是**目前 Web 3 的孵化器不会提供任何线下的办公场地**。结合 Web3 的行业特征和过去三年的远程办公状态，可以看到现在 80% 到 90% 的 Web 3 团队都是分布式的 global 团队。

  


三是**经济模型在 Web3 项目孵化中有很重要的地位**。对 Web2 出身的团队来说，可能更擅长软件设计，提升用户体验和用户路径等等，对经济模型以及经济模型的生命周期设计也不太熟悉。而对 Web3 中技术出身的团队来说，也需要这块的知识补充。

  


  


**03**

**Web3GO Labs**

**为什么选择做孵化器？**

**1**►

**现状&困境**

**- 行业周期**  


Web3 行业的周期性非常短，这意味着创业者或早期团队必须在短时间内试错，要求动作更快、执行力更好。当然，前提是选择正确的方向。如果选择正确方向并快速执行，就可以带来指数级增长，反之则会非常困难。

  


**- 融资市场**

当下 Web3 处于初期阶段，非常类似早期的互联网，存在着草莽、无序、安全性差、合规法律不健全等问题。资本对于它的认知以及出手的态度，也会根据行情的情况摇摆。

  


**- 变现模式**

当下 Web3 项目的变现模式不清晰，目前这个行业里大部分的商业模式还是无法被变现的。很多团队还需要靠融资去推进自己的产品，或者是靠 token listing 去实现一部分的变现，总体上还没有出现稳定的变现模式。

  


**- 冷启动**

同时，因为 Web3 还在早期阶段，做项目或搭建软件产品时冷启动的难度很大。在移动互联网行业中，冷启动是不存在的，因为我们有各种各样的资源和流量库，但目前 Web3 的用户容量很小，也没有什么流量采买的渠道，主要还是要靠团队自身去探索引流。

  


在大量田野调查后，结合对 Web3 行业现状和早期项目共性问题的判断，Web3GO Labs 团队搭建了这个孵化器。

  


那么，具体能提供什么孵化服务？

  


**Web3GO Labs 选择从早期 Web3 创业团队的需求出发。**

  


**2**►

**需求**  


**- 天使投资**

对于早期的创业团队来说，天使投资人能提供一定的资金，帮助团队实施早期验证，以确定他们的想法是否可行。这些资金可能规模不大，但它们足以使团队开始行动，并为未来的融资做好准备。此外，天使投资人也能够帮助团队建立联系人网络和获得资源，以支持其成长和发展。

  


**- Web3 思维和玩法**

许多 Web 2 团队正在转型到 Web 3 领域。然而，这两者之间存在巨大的思维和玩法差异。 Web 3 包括 DAO、激励机制和投票选举等特殊元素，这些元素甚至会涉及到社会行为等方面的思考研究。因此，对于新进入 Web 3 领域的团队来说，需要快速学习、理解这些特殊元素，并应用到自己的项目中。

  


**- 冷启动**

在移动互联网时代，许多团队通过用户和流量来启动产品。然而，在 Web 3 时代，这种思维模式不再适用。团队需要思考如何在零流量的情况下，自己的产品能够启动并跑起来。

  


**- 需求证伪**

需求证伪是很多早期团队容易忽略的部分。很多团队往往会忽略用户对产品的真实需求，只是根据自己的假设，花费大量时间和金钱去开发一个并不被市场需要的产品。为了避免这种情况，团队需要采取客观的方法，去验证目前产品输出的功能是否符合用户的需求。

  


**3**►

**服务**  


目前，孵化器会在早期轮次投资一些团队，也会向处于创意或 MVP 阶段的团队提供咨询和合作服务。至于寻找融资方面，目前这不是 Web3GO Labss 的核心服务，大部分情况下会婉拒。

  


总体上，**Web3GO Labs 孵化器有两个重要模块：**

  


第一个模块是**产品需求证伪**。如果一个项目进入到孵化期，会要求团队首先完成这个模块。孵化器会帮助团队做好充分的市场调研，确保他们的产品能够真正满足市场需求。

  


另一个模块是**市场策略**。孵化器会结合项目的技术架构等特点，帮助团队选择适合自己的网络和策略，确保产品能够得到更好的推广和支持。

  


**04**

**观点**

**1**►

**从 0 到 1 构建首个 Web3 项目**  


对于搭建自己的第一个项目，很多人会有一个共同的思路，即自己作为用户，发现了某个产品的痛点，然后想办法去解决它。当然也会有专家型选手，在搭建项目前对赛道做一个彻底的调查。**但无论 idea 从何而来，在构建项目前，建议先考虑它在整个生态或产业链中的位置，这将决定项目未来的天花板。同时，在开始搭建产品前，应先花时间做需求证伪。**

  


在搭建 MVP 的过程中，**要注意两个误区：想的太多做的太少，或是完全不想一直做**。不要在臆想的需求中搭建自己的理想国，而是应该不断地与目标用户沟通，抓取用户的真实需求和产品反馈。也不要陷入完美主义，在 MVP 版本开发时，更应该专注于产品的核心功能。

  


此外，在项目初期，**如何获取种子用户是非常重要的**。一种常见的方法是通过 KOL 矩阵或广告投放来获取流量，但是实际上通过这种方式获取的流量质量如何，有待商榷。因此，Web3GO Labs 提供了另一种思路：**先通过 B2B 合作来获取更多的社区关注度**，与其他项目或组织合作共同推广。这种方法可能需要一些时间和谈判，但可以为项目带来更多的流量和忠实用户。

  


另外一个重要的部分，也是很多 Web 2. 0 的开发者进入 Web 3 搭建早期产品时容易忽略的点，就是**选择一条适合自己的网络，建立自己在这个网络中的生态位**。如何选择自己网合适自己的网络，这对团队来说很重要，需要详细的研究调查。

  


**2**►

**Web2 大厂人入局优劣势**  


随着 Web3 逐渐崛起，许多 Web2 大厂人开始转型入圈。这是个好的趋势，因为像任何一个行业的兴起一样，劳动力是先行的。因此，当有大量的技术人员进入一个行业时，这个行业一定是快速发展并即将迎来爆发的阶段。

  


然而，这个转型过程中， Web2 背景有优势，也有不足。

  


Web3 是一个重软件型的行业，需要大量的软件人才，而大厂出身的工程师往往有着很扎实的代码能力。

  


然而， **Web3 和 Web2 互联网的玩法和逻辑有所不同**，习惯了传统营销和运营的人，需要一定的时间来适应 Web3 的路径和打法。同时， Web3 最有趣的特点之一，就是它的快速变化，可能今天是这样的打法，明天就会是那样的打法。Web3 的变化速度可能比移动互联网更快，因此对于大厂人来说，需要具备快速适应的能力。

  


总的来说，**现在转型去 Web 3 是否合适，取决于个人的背景和经验**。如果一个人在 Web 2 工作过且具有 0-1 产品搭建和推广的能力，那么适应 Web 3 的速度很可能会很快。同样地，有创业经验的人也可能非常适合 Web 3。然而，对于那些在 Web 2 大组织内待了很长时间，并且缺乏以上经验的人来说，转型到 Web 3 可能需要更多的慎重考虑。还是要从个人优势和特长出发，也可以先以 part-time 的方式，尝试探索 Web 3 的机会和可能性。

  


**3**►

**当下资本市场情况如何？**  


目前来看，机构会比去年后半年相对更积极一些，但是今年截止目前，早期团队在融资上还是会比较困难。大部分的机构更倾向于看而不是投，更愿意 follow，而不是去 lead 一个项目。所以每一个项目都需要去找到自己的领投方，才有可能把这一轮次推进下去。
