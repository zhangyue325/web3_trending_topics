1. Getaverse DID 平台
-------------------

创建一个用户去中心化的 DID 身份信用管理平台。该平台引入 Web3 概念的热门项目。项目方享受平台提供的技术服务支持，帮助项目的推广，并协助项目方标注自己的身份、权利、积极性、资质证书等。用户可以通过平台创建专属的去中心化的 DID 身份，并以该身份参与各种项目中。

![](https://img.foresightnews.pro/202303/1295c4398cb5d629c5c419c76d50b361.png?x-oss-process=style/scale70)

2. 流量共享与热点追踪
------------

Getaverse DID 对平台上的项目进行区分，大致可分为 DeFi、GameFi、DAO、DEX、Layer1、Layer2 等几类。不同类型的项目会吸引不同的用户群。当用户聚集在平台上时，巨大的用户流量池会增加平台上项目的曝光率。对于 Web3 用户来说，通过观察平台上项目的互动数据，可以准确捕捉到当前的热门项目，并快速参与其中。

3. SBT & GOAT（参考 GALXE 项目）
--------------------------

Getaverse DID 基于 ERC721 和 ERC5114 协议开发了 SBT 和 GOAT，用于记录用户的链上和链下互动行为、社交行为、资产证明、身份证明、享受权利、空投代币等。

对于 Getaverse DID，平台提供身份、教育、工作等信息的权威认证。由于 SBT 的不可转让特性，当用户完成相应的信息认证后，可以领取平台发放的 SBT，进一步建立自己的去中心化 DID 身份。

对于项目方来说，Getaverse DID 平台提供了 SBT 和 GOAT 铸造服务。项目方可以根据自己的需要编辑相应的活动信息。当用户完成条件后，将通过发放 SBT 和 GOAT 对目标用户进行标记。

4. 用户隐私数据保护
-----------

由于用户与 Getaverse DID 平台的交互过程中涉及到一些隐私数据，例如在身份认证时需要用户提供身份证信息。为了保护用户的隐私数据，Getaverse DID 计划与权威的第三方认证机构合作。所提供的信息将被审查，平台只接收审查结果并做出后续回应。用户可选择是否保存私人数据供后续使用。保存的私人数据会用非对称加密算法进行加密，并存储在一个分散的存储应用中。用户负责私钥，所有权限属于用户。

5. 去中心化的社会生态（参考 LENS 协议）
------------------------

Getaverse DID 计划在平台上建立完善的社交设施，如讨论组、话题栏等。项目方可以利用该设施服务建立官方群聊，实时更新项目动态，并通过发布话题引发用户讨论。我们会筛选用户的建议，以优化项目的未来发展。对于用户来说，在各种社交媒体上收集项目信息的动作大大减少，更方便、更直接地咨询相关项目信息，在讨论组和话题中交流意见，找到志同道合的朋友。用户的每一个社交行为都以分散的形式被记录下来，将来也会被纳入信用评价的参考项目中。

6. DID 身份信用评估
-------------

Getaverse DID 提供的各种基础服务最终都指向用户 DID 身份的信用评价，信用以积分形式显示。.0 地址信息、平台 SBT&OAT 数据、平台去中心化社交数据，进行全方位的分析和评估，给用户 DID 身份一个权威可靠的信用分。用户在 Getaverse DID 平台上的所有互动，都相当于一个信用积累和建设的过程。

7. DID 身份的多链共享
--------------

Getaverse DID 身份协议被部署在多个 EVM 网络中，如 ETH、BSC、Polygon 等。用户的 DID 身份信息、信用评分、互动行为等数据都与协议紧密相连。Getaverse DID 将实时同步各链之间的智能合约数据，DID 身份互通不需要支付任何气体费用，从技术层面真正实现用户 DID 身份的多链共享。

8. 外部应用
-------

1. ABI（智能合约接口）项目方可以通过查询 Getaverse DID 在链上发布的智能合约来确认用户的 DID 身份信息和信用，并可以有针对性地设置项目的参与门槛或设置股权分类。他们还可以根据查询结果制定一系列可扩展的规则颁布和约束。用户端可以通过查询结果判断其他用户的身份和信用程度，并在有资金交易时做一定的参考。
2. API（服务器接口）Getaverse DID 为开发者提供了全面的 API 文档。开发人员可以根据 API 文档中各种功能的编程和应用方法，进行个性化的定制设计。对于 Getaverse DID 提供的数据，通过开发人员的整合和应用，可以创造出更有价值的东西。
3. Web2.0 与 Web3.0 应用账户之间的互操作性当 Getaverse DID 的用户身份被市场 / 机构 / 团体认可后，它将努力与各个层面的应用开展深入合作。用户的去中心化 DID 身份可以作为媒介。DID 身份相当于一个账号，可以一键下载和登录社交媒体应用，如 Twitter、Discord 等。未来，拥有 DID 身份的用户可以享受一键下载 / 一键登录知名交易所，作为真正的交易者投资于加密领域。
当用户的 DID 身份应用于 Web3 领域时，使用 DID 身份参与热门 Web3 项目时，可以享受项目更多的信用权益，如与去中心化金融 DeFi 项目合作，DID 信用身份可以享受更低的贷款利率或 Fee 减免等权益。
