**前言**

Coinbase 正大刀阔斧地建设链上基础设施，近日宣布基于 OP Stack 构建一条二层 Rollup 链——Base, 一时间 Crypto 市场上掀起了“Bullish on Rollup” 的呼声。

Coinbase 作为全美第一大加密交易所，此番举措或将成为 Rollup 赛道发展、革新的催化剂。

  


**背景**

以太坊网络性能的局限性催生出了多样化的 Layer 2 扩容解决方案，Rollup 方案作为其中的翘楚领跑整个扩容赛道。以太坊通常被看成一个单体区块链网络，单体中主要由 3 大功能层级构成：数据可用性和共识层、结算层和执行层。

由于区块空间是有限的，所有功能层级在单体中运行消耗了大量的链上资源，导致网络拥堵、链上 gas 费用飙升。Rollup 本质上是将执行层从单体区块链中分离出来的模块化扩容解决方案，将复杂运算转移到链下，再把链下执行结果传输到链上验证，在继承以太坊安全性的情况下大幅提高网络的性能。

据 L2BEAT 数据显示，截止至 2023.03.02，Arbitrum TVL 为 34.3 亿美元，份额在整个 Layer 2 赛道中占比高达 54.25%；Optimism TVL 紧随其后，两者 TVL 总和已超过 80% 的市场份额。Layer 2 赛道竞争格局看似已由 Arbitrum 和 Optimism 捷足先登，实则风起云涌，越来越多的 Rollup 方案如雨后春笋般涌出，带着它们的创新叙事，企图蚕食一定的市场份额。

![](https://img.foresightnews.pro/202303/0a18736d6e6a24ae93bfb3f7cac21ffb.png?x-oss-process=style/scale70)而其中最引入注目的，是由此衍生出的一个被称为“Rollup as a Service”的全新赛道。

**顾名思义，RaaS 代表把 Rollup 的部署作为一种服务，为开发者从 0 到 1 构建 Rollup 提供了极大的便利。**

RaaS 理念的雏形最早可以追溯到 2021 年 Celestia 的 Optimint( 现更名为 Rollkit), 旨在为开发者提供一整套通用软件开发套件 (SDK). 由 Optimism 团队提出的 OP Stack 堆栈是这一赛道的代表性项目，近期 Coinbase 基于 OP Stack 构建其 Optimistic Rollup(ORU) Base, 这是一个起点同样也是一个契机，点燃了我对 RaaS 赛道的热情和憧憬。

在讲 RaaS 赛道的具体项目之前，先来捋一捋这个赛道之所以出现、兴起或者说即将兴起的原因和驱动力。

**我始终认为最核心的驱动力是需求：**以太坊性能的局限性催生了扩容的需求，所以有了 Rollup; 从 0 到 1 部署一条链并非易事，如果有通用的工具会事半功倍，因此出现了 RaaS. 当然，上述观点成立的大前提是 Rollup 本身具有需求，目前来看，通用型 Rollup 的需求已经得到市场的证明；特定应用型 Rollup 的需求还有待于市场验证。

  


**一、RaaS**

  


除了上述所说的 Optimism 的 OP Stack, Celestia 的 Rollkit, RaaS 赛道包含的项目还有很多。从是否需要编写代码角度，RaaS 项目可分为 2 类：SDK 和无代码 (No-Code) 方案。SDK 方案即为开发者提供一整套通用的软件开发套件，让部署 Rollup 和部署智能合约一样简单；无代码方案，顾名思义，即无需编写一行代码部署 Rollup( 一键发链 ).

其中 OP Stack 和 Rollkit 均属于 SDK 方案，Dymension, Sovereign, Stackr 同属这一范畴。无代码部署 Rollup 的项目包括 AltLayer, Saga, Eclipse 和 Caldera. 这里不会花太长的篇幅对 RaaS 赛道的每个项目进行详解，仅做简答介绍。

文章更多侧重于我自己对 RaaS 赛道的一些思考，希望能起到一点抛砖引玉之效。

![](https://img.foresightnews.pro/202303/2a10a156ef0ad630eabc8df8c6f7cc04.png?x-oss-process=style/scale70)  


**SDK**

**1. OP Stack**

OP Stack 是 Optimism 团队为了实现模块化区块链架构的软件堆栈，将区块链网络分为共识层、结算层、执行层 3 大功能层级。通过将不同功能层级解耦，再以 API 软件堆栈的形式把所有层级组合起来，极大地简化了模块化区块链构建的流程。

OP Stack 给 Rollup 开发者带来了更大的自主性，能够切换、替换、修改不同模块层级，就像切换 API 一样。Optimism 团队提出了他们认为的区块链终局形态“超级链”，即由一组高度集成的 OP Stack 链组成，链之间通过共享消息传递格式进行通信，实现 Rollup 链间互操作性。

**2. Rollkit**

Rollkit 是由 Celestia 社区开发的模块化发链 SDK, Rollkit 相对于 Rollup 就相当于 Tendermint 相对于 Cosmos Layer 1. 开发者可以基于 Celestia 提供的数据可用性 (DA) 和共识层构建 Rollup, 进一步提高可扩展性。另外，Rollkit 也带来了可定制化的优势，DA 层、排序器、执行环境、证明机制等都可以定制。

![](https://img.foresightnews.pro/202303/4af7374e129986162d8fe792c5c37991.png?x-oss-process=style/scale70)**3. Dymension**  


Dymension 的 RollApp 开发套件 (RDK) 的功能与 Cosmos SDK 类似，帮助开发者在 Dymension 结算层上部署 RollApp( 基于特定应用的 Rollup). 结算层扮演的角色是维护所有 RollApp 的安全性，也被称为 Dymension Hub, 换句话说，所有基于 RDK 构建的 RollApp 共享 Dymension Hub 的安全性。值得注意的是，Dymension 在结算层中引入了 RollApp 虚拟机 (RVM), RVM 支持全新的争议解决机制，因此可以支持各种不同的执行环境。

![](https://img.foresightnews.pro/202303/fb26608a1b0f4fa69065507d8763acfa.png?x-oss-process=style/scale70)4. **Stackr**  


Stackr 是以太坊基金会主办的 Layer 2 Community Grants 2022 22 个获奖项目之一，它在 130+ 项目中脱颖而出。旨在通过 Stackr SDK, 让开发者更便捷地部署可定制化的 App-specific ORU.

5. **Sovereign**

Sovereign 相对于以上 4 个 Rollup SDK 项目而言稍显“另类”，上述 SDK 方案都是以构建基于欺诈证明的 ORU 为切入点，而 Sovereign 一开始便专注于基于零知识证明的 zk-rollup(zkRU) 的构建。零知识证明是密码学中非常难的一个分支，zk-rollup 的开发要求开发者拥有很扎实的密码学功底。Sovereign 提供的 SDK 能够让没有密码学专业背景的开发者也能使用零知识证明技术开发公链，让开发 zk-rollup 跟开发 Dapp 一样简单。

  


**No-Code**

**1. AltLayer**

AltLayer 是具有代表性的无代码 Rollup 解决方案之一，旨在解决单一热门 Dapp 造成整个网络拥堵的问题。由于部署在同一区块链网络的不同 Dapp 共享有限的区块空间，当特定 Dapp 链上交易激增 ( 如限定时间的 NFT 铸造活动等 )，会造成整个网络拥堵 gas 费用飙升。AltLayer 通过提供部署 Flash Layer 提供一次性执行层，为 NFT 铸造事件提供专用的区块空间，与链上的其他项目完全隔离，在铸造活动结束后关闭执行层，进而提高网络的可扩展性。

**2. Saga**

Saga 是部署在 Cosmos 生态上的项目，愿景是通过横向扩容打造千链生态。Saga 的架构跟上述 SDK 方案 Dymension 类似，Rollup 在 Saga 架构中被称为 Chainlet. 由于基于 Saga 主网构建，所有的 Chainlet 共享 Saga 主网提供的安全性。通过共享安全，Chainlet 无需原生代币即可运行。由于链游底层基础设施的完善和扩容成为当下亟需解决的问题，Saga 最初将专注于游戏、娱乐链等 Web 3 基础设施。

**3. Eclipse**

Eclipse 旨在为任意 Layer 1 提供通用的结算层，Rollup 再基于 Eclipse 的结算层构建。Eclipse 为开发者提供了一个高度可组合的 Rollup 框架，可以自主选择虚拟机、数据可用性层等。目前，Eclipse 支持的 Layer 1 包括 Celestia, Polygon 和 EigenLayer, 后续将支持更多的 Layer 1 网络。

![](https://img.foresightnews.pro/202303/0ffa6e0bde6a62e156ba9d0e63144f9a.png?x-oss-process=style/scale70)**4. Caldera**  


Caldera 支持“一键发 ORU 链”，无需编写一行代码，将开发一条 Rollup 的时间缩短至分钟级别。Cladera 链模块化堆栈由 3 部分组成：Caldera Chain( 执行层 )/ EVM 结算层 / 数据可用性层 (DA). 它能基于任何 EVM 兼容链作为结算层，Caldera 的执行层是基于欺诈证明的 ORU, 即通过运行欺诈证明机制维护 Rollup 网络的安全性。

![](https://img.foresightnews.pro/202303/7f542b44a44e844fa0a0eddce7817bc3.png?x-oss-process=style/scale70)  


**二、Rollup 模块化架构**

  


从 Rollup 的模块化架构进行划分，大体可以分为 3 类：

**1）通用智能合约 Rollup;**

即 Layer 1( 如以太坊 / Polygon 等 ) 负责结算 + 共识 + 数据可用性，执行层 (Rollup) 从单片链中分离出来，代表性例子如 Optimism/ Arbitrum.

**2）结算 Rollup;**

即 Layer 1 负责共识 + 数据可用性，结算层作为独立的层级分离出来，Rollup 基于结算层构建，由此构建的 Rollup 共享结算层。结算 Rollup 根据功能不同又可以划分为特定应用结算 Rollup 和通用型结算 Rollup. 如基于 Dymesion Hub 构建的 RollApp, 基于 Saga Chain 构建的 Chainlet.

**3）主权 Rollup;**

即 Layer 1 负责共识 + 数据可用性，Rollup 负责结算 + 执行，由于 Rollup 可以决定其自身交易的有效性，所以我们认为它是有“主权”的。与结算 Rollup 类似，根据功能不同主权 Rollup 又可划分为特定应用主权 Rollup 和通用型主权 Rollup. 如基于 Celestia Rollkit 构建的 Rollup.

更具体地，Rollup 可以分为 5 类：通用 Rollup, 特定应用 + 通用型结算 Rollup, 特定应用 + 通用型结算主权 Rollup. 如果再跟证明机制 ( 零知识证明 / 欺诈证明 ) 组合，可以衍生出 5 × 2 = 10 种 Rollup 组合。再具体一点，开发者可以选择 Celestia/ Eigen DA 做数据可用性和共识层，Dymension/ Saga/ Eclipse 做结算层，构建 Rollup. 随着模块化项目的增多，由此形成的模块化 Rollup 组合甚至可以达到成百上千种，而所有的 Rollup 组合，都能通过 RaaS 相关项目提供的基础设施轻松便捷地部署，RaaS 赛道所散发出的潜力是不容忽略的。

![](https://img.foresightnews.pro/202303/e575a861773ebcbb7c56194c6bf695c6.png?x-oss-process=style/scale70)  


**三、看好 RaaS 赛道的一些想法**

  


之所以看好 RaaS 赛道，主要基于以下的一些投资逻辑：( 仅做观点交流，没有任何投资建议。DYOR. NFA)

**1) RaaS 相关项目备受资金青睐；**

如 Coinbase 基于 OP Stack 做 Rollup; RaaS 赛道相关项目获得一众明星 VC 投资，如 Caldera 获得红杉、蜻蜓资本领投的 900 万美元融资；Sovereign 获得 Robot Ventures, Maven 11 等 VC 740 万美元融资；Eclipse 完成两轮共计 1500 万美元的融资，投资机构包括 PolyChain, Tribe Capital 等……

**2) RaaS 带来一种范式的模块化区块链底层创新；**

作为核心底层基础设施，RaaS 的完善将对区块链网络架构的革新带来深远的影响。RaaS 的普及极大地降低了开发者从 0 到 1 构建一条 Rollup 的门槛，准入门槛的降低将带来激励的竞争，能够很好地诠释优胜劣汰的“丛林法则”，进而倒逼 Rollup 的创新。

**3) Rollup 赛道尚未完全爆发，RaaS 赛道更是处于早期。**

虽然目前来看，以 Arbitrum 和 Optimism 为代表的 ORU 占据了该赛道超过 80% 的市场份额，但 zk 系的相关项目还处在蓄势待发的状态。zkSync 和 StarkNet 虽然都已上线主网，但上线时间不长且暂未迎来链上生态的大爆发。因此，现在说 Rollup 赛道大局已定还为时尚早。

当然，机遇和挑战是并存的。无论是 Rollup 还是 RaaS, 都存在很多悬而未决的问题。如排序器的中心化 ( 虽然目前如 Rollkit/ Optimism/ Dymension 等都有提出去中心化排序器的解决方案，但目前都没有落地 )、流动性碎片化、ORU 的挑战期过长等问题都有待解决。虽然存在一些问题，但都有与之对应的解决方案，只需重点关注是否能落地以及落地的效果即可。

  


**# 结束语**

  


早在公链大规模爆发之前，人们热议的焦点是未来的公链格局到底是单链还是多链。至少从目前来看，“Bullish on Multichain” 的人更胜一筹，我们除了以太坊之外，还看到了 Polygon/ BNB/ Cosmos 等 L1 都在冉冉升起。

而对于 Rollup, 我们面临同样的一个问题是：不远的未来，Rollup 的格局到底是单个 Rollup 一家独大还是 Multi-Rollup 竞相繁荣？我个人更期待看到的是 Multi-Rollup 的格局，借助 RaaS, Rollup 在竞争中发展，不断推陈出新。但最后谁会在激烈的竞争中存活，谁又会永远地退出 Rollup 的历史舞台？没有人知道，只有时间能告诉我们答案。

毕竟，只有当大潮退去，才知道谁在裸泳。
