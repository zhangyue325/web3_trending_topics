区块链的去中心化、不可篡改和透明的特性使得它可以应用于许多领域，如 CBDC、产权保护、供应链管理同时也包括了很重要的数据存储等。

区块链存储技术提供了一种去中心化的、高效的、低成本的数据存储解决方案。目前已有 FIL, AR, Storj 等类型的项目，在 2020 年的热点分析文章时，曾举例介绍了这部分区块链存储项目。

而 BNB Greenfield 是一种分布式存储服务生态，它提供了一种可扩展、高效、低成本的存储解决方案，**本文将探讨 BNB Greenfield 在区块链存储领域的价值、市场潜力、与其他存储项目的对比，和对 BNB token 价值的影响。**

一、BNB Greenfield 基础架构
---------------------

此系统设计前提假设：

假设 1：大多数数据提供者和区块链验证者都是相对合理的实体和个人。他们正常不会作恶，因为他们罪恶成本远大于利润。

假设 2：服务提供者和消费者都希望实际“合同”允许有限责任并提供退出选项。

假设 3：数据具有价值，用户希望通过智能合约自动化、隐私和透明来提取这种价值。这使得 BNB Greenfield 和 BNB 智能链（BSC）之间的跨链设计相当可观。这是最重要的假设，并有望接近事实。

#### ● **狭义的 BNB Greenfield：（最容易理解）**

![](https://img.foresightnews.pro/202303/6cf4d4173732051ec1a9321bf404c00a.png?x-oss-process=style/scale70)

**这个干活的公链叫 BNB Greenfield Blockchain,** 简称（GF）

用来记录存储共识元数据， BNB 作为其 gas 和治理的原生代币，有自己的治理逻辑。Storage Provider（SP）提供存储服务及扮演相应角色。使用 GF 公链记录。每个 SP 都能够响应用户的写入（上传）和读取（下载）数据的请求。

#### ● **完整的 BNB Greenfield 体系：（加入生态互通与协作共赢）**

![](https://img.foresightnews.pro/202303/3033f7ff10309c647df299b8427b9cc0.png?x-oss-process=style/scale70)

如图所示，完整的 BNB Greenfield 系统，加入了已经较为成熟的 BSC 公链与 BN 社区用户的互通，当用户想要在 Greenfield 上创建和使用数据时，可以通过 BNB Greenfield dApps（去中心化应用程序）与 BNB Greenfield 核心基础设施进行交互。GF 链和 BSC 之间有一个原生的跨链桥。

BNB Greenfield 链使用基于 Tendermint 共识的 POS 机制来确保其自身的网络安全，每 2 秒创建一次块。此处，GF 链的验证者（Validator）和 SP 里面的职责并不相同。

* **账户模块：**用来管理用户的账户信息，包括地址、余额等。
* **交易模块：**则负责处理用户之间的交易信息，包括交易金额、fees 等。
* **合约模块：**允许用户通过智能合约实现自动化的交易和业务逻辑，同时也可以用来实现去中心化的应用程序。
* **应用生态：**包括开发和支持各种应用程序和 DApp（去中心化应用），为用户功能和服务。
* **存储节点**：存储模块节点则用来存储区块链上的交易信息和账户信息等数据。节点负责存储和检索数据的基础设施。存储节点可以通过多种方式参与网络，例如使用 Binance 托管的云存储服务、自行搭建存储节点、或使用第三方存储服务提供商。
* **区块链网络**：是用于记录存储节点和用户之间交互的信息，例如存储和检索请求、奖励机制等。BNB Greenfield 存储项目是基于 BSC 开发的（EVM 兼容），验证者选举和治理基于提议投票机制，是基于 Cosmos SDK 的治理模块进行了修订，任何人都可以创建和提议成为验证人。
![](https://img.foresightnews.pro/202303/c2c06b34ebd11e359efc89eee0fd0e88.png?x-oss-process=style/scale70)

二、BNB Greenfield 的经济模型
----------------------

BNB 作为 BNB Greenfield 链的原生代币，具有多种用途，包括支付手续费、参与节点选举、购买 DApp 等。BNB 的总流通量将保持不变，而是在 BNB Beacon Chain、BSC 和 Greenfield 之间流动，起初的 BNB 通过 BSC 锚定数量，并跨链到 GF 链上。

![](https://img.foresightnews.pro/202303/2bc75e34305bd69f577e6c1a2aed3c59.png?x-oss-process=style/scale70)

  


* **质押类业务与应用消耗：**保证金、event 活动等质押消耗场景
* **存储服务费**：用户在使用存储服务时需要支付一定的费用，这些费用可以使用 BNB 进行支付。
* **存储节点奖励**：存储节点提供高质量的存储服务后可以获得 BNB 奖励。
* **存储挖矿奖励：**用户可以通过上传数据到存储网络中，获得 BNB 奖励。
* **节点参与奖励：**存储节点参与存储网络的维护和管理，获得奖励。
其次，数据权限可以跨链转移到 BSC 上，成为 BSC 上的数字资产。这将使这些资产与 BSC 上现有的 DeFi 协议和模型整合创造了多种可能性。而每一次交互与创造组合也会促进 BNB 的使用场景，释放一些潜力。

![](https://img.foresightnews.pro/202303/6b20c4fbf9753d7a83ae8689f3cb7bb6.png?x-oss-process=style/scale70)

  


BSC 上的智能合约进一步增强了这一点，它与 Greenfield 区块链上的账户享有相同的地址格式，并且可以成为数据对象的所有者并继承不同的权限。这将释放出基于数据的新商机。

三、BNB Greenfield 的市场潜力：
-----------------------

1. 目前全球存储数据的需求正在不断增长，特别是在数字化和云计算时代，存储需求量已经成倍增长。虽然 Web3 去中心化存储目前还没有开始需求井喷，但凡事需要未雨绸缪。元宇宙，各类资产通证化之后的存储，数字存证等一旦开始大规模普及，到时也会产生巨额的需求。区块链技术引导的存储，可以更好地满足用户的需求，特别是在数据安全和可靠性方面。

2. BNB GF 直接拥有 Binance 现成的用户群和社区共识。这意味着 Greenfield 存储项目至少可以得到更多的用户和社区的支持与知识的科普，使其在市场上具有更高的知名度和相应的竞争力。

3.BNB Greenfield 存储项目采用了较为稳妥的共识机制可以让用户相对更加信任网络，并且由于生态互通性与 EVM 自动兼容性，也进一步使 BNB Greenfield 存储项目在市场上拥有相对后来者优势与竞争力。

4. GF 为开发者提供完善的开发工具和模块，这也可以吸引更多的开发者参与到 BNB Greenfield 区块链生态系统的建设和发展中与后续的交互之中。同时，跨链的模式，使得项目方，开发者可以产生较快的在生态内进行切换，这种优势，一定程度上提升了吸引力与竞争力。

个人认为，BNB Greenfield 存储项目，作为一个新兴的区块链存储生态，具有值得期待的市场潜力与足够大的想象空间。它可以满足用户对于安全、可靠、高效、低成本存储服务的需求，并为整个生态系统带来更加可持续的发展和创新。

然而，任何事情成功都是多方面因素的，GF 也需要面对一些挑战和风险，如突发监管，全球性黑天鹅事件，市场需求、技术创新、竞争对手等。该领域目前存在较多竞争者，如 Filecoin、Storj 等项目，BNB Greenfield 需要在技术、安全性、用户体验等方面不断改进和提高，以确保其竞争力和市场份额与可持续发展。

五、BNB Greenfield 主要潜力案例 **( 官网 )：**
-----------------------------------

![](https://img.foresightnews.pro/202303/7ff20166d5015ef99e2163939d01c3dd.png?x-oss-process=style/scale70)

**1. 网站托管**

由于 BNB Greenfield 提供 api 并使用与 Amazon S3 非常相似的概念，用户可以通过 BNB Greenfield 轻松部署他们的网站，并方便地管理其通过 BNB 的支付。

**2. 个人云存储**

用户可以使用 BNB Greenfield 创建自己的网络驱动器，通过桌面和移动设备使用私钥上传和下载加密文件、照片和视频。

**3.区块链数据存储**

Layer1 包含数十 TB 的历史数据，其中大部分已无用或处于休眠状态。BNB Greenfield 可用于存储这些数据，减少 L1 延迟并提高数据可用性，以及在需要时无缝地将死数据或休眠数据传输到 L1。BNB Greenfield 还可以用作存储 L2-Rollup 交易数据的更便宜的选择。

**4. 内容发布**

作者可以将他们的作品存储在 BNB Greenfield 上，并将它们镜像到 BNB Smart Chain 上，在那里他们可以在收到付款后通过授予买方地址的阅读权限来出售它们。其他书面数据如历史统计数据、科学论文等的提供者也可以这样做。BNB Greenfield 目前没有反盗版设计。作者、发布者和 dapp 应该意识到这些限制，并在选择使用 BNB Greenfield 时自担风险。未来 BNB Greenfield 可能会引入更多与版权相关的功能。

**5. 社交媒体**

BNB Greenfield 为社交媒体影响者和关键意见领袖提供了一个最佳的基础设施，以完全拥有自己的内容和数据。付费后，dapp 仍然可以使用这些数据——包括内容本身和相关的社交图——并通过去中心化的社交网络将其发布给公众。BNB Greenfield 还使创建者可以很容易地利用基础设施的独特许可模式为他们的粉丝创建私有子组。

**6. Token 管理注册中心**

BNB Greenfield 为通证管理列表提供了一个完美的解决方案。BNB 智能链上的智能合约可以承担注册的完整治理和激励逻辑，同时拥有在指定桶下为业务创建每个列表对象的权利。不可变的列表对象和完全跟踪的变更历史是这种去中心化投票和治理思想的无价功能。

**7. 个人数据市场**

个人数据，比如页面浏览量、注册量、点击量、行为数据等等，需要防止平台 ( 通常是大型的、集中的垄断企业 ) 滥用这些数据。许多法律被定义为惩罚滥用，但很少有积极的解决方案。BNB Greenfield 将允许应用程序将用户的加密个人数据存储在您自己的个人帐户中，并且只允许其他应用程序在您授予许可的情况下使用您的个人数据——或者更好的是，个人可以进行收费。

六、BNB Greenfield 存储项目对 BNB 的价值影响：
---------------------------------

个人认为，BNB Greenfield 存储项目从立项来说，客观层面上的确可以对 BNB 的价值产生相对积极的影响，但具体的影响程度还需要考虑许多因素。

1. BNB Greenfield 将进一步增加 Binance Smart Chain 的使用率和认可度，越来越多的项目和用户选择使用 BSC 从而提高 BNB 的整体需求和价值，通过跨链的切换与整体生态系统的互通，BNB 可以被更广泛地使用于生态系统中的各种应用。成为一种多赢互补系统。

2.BNB GF 可以为 Binance 提供可持续性与创新性的后续增值服务，这可以增加 Binance 的收入和用户数量，提高 Binance 的盈利能力和市场价值，进而提高 BNB 的价值。

3.BNB GF 拥有 Token 经济模型，这可以吸引更多的存储节点和用户参与，使得 BNB 的需求量增加。同时，还可以为用户提供与节点提供相对更多的灵活支付方式。

从长期的角度来看，BNB Greenfield 存储项目也可以为 Binance 智能链带来更多的应用场景和用户，从而进一步提高 BSC 的整体使用率和价值。如果 GF 能够成功运营，并运行良好，那么它可能会对 BNB 整体市值产生积极的影响，但具体影响的多大还需要根据项目的实际进展情况不断优化评估。

然而，特别是作为一个三位一体的系统来说，一荣俱荣，一损俱损，有时外界因素的出现，也会导致短暂的恐慌或 FOMO，这也是我们不能直接定义 BNB Greenfield 对 BNB 真实影响的原因之一。
